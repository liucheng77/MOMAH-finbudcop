# 财务与预算 AI 智能体演示（Financial & Budgeting Copilot）产品需求文档（PRD）

> 适用对象：MoMAH（沙特市政与住房部）/ Balady 财务与预算业务场景
> 文档性质：基于当前 Demo 实际实现如实描述；Demo 为原型演示，数据为合成数据，AI 过程为脚本化演示（非接入真实大模型 / 真实业务系统）
> 版本：对应应用内 Release Notes v0.9（2026-06）

## 一、产品概述

### 1. Demo 整体定位、核心用途与解决的业务问题

本 Demo 是一套面向财务与预算全业务总局的 **AI 智能体协作（Agentic）演示原型**，以「编排器 + 多智能体 + 人工审批」的方式，演示 AI 如何在财务与预算工作中提供决策支持。

核心定位是 **决策支持，而非报表自动化**：AI 负责把分散在多个源系统的数据统一、识别异常与偏差、生成分析与草稿，但关键动作始终交由业务人员审批（Human-in-the-loop），AI 不直接对外发布或落账。

它要解决的业务问题包括：

- 财务数据分散在多个源系统、口径不一，人工汇总与对账成本高；
- 预算执行滞后、超支、开票/征收缺口等问题发现不及时；
- 跨部门（收入征收、资产、成本、合规、审计、报告等）协同链路不清晰，业务流转难以追踪；
- 分析与报告高度依赖人工，领导决策所需的「一句话结论 + 可执行下一步」产出慢。

Demo 通过统一的部门工作台、贯穿 G-06 的端到端业务故事线、各总局的多智能体编排流程，以及可一键直达的「业务广场」，把上述问题以可视、可交互的方式呈现给客户。

### 2. 目标使用人群、整体功能简述

**目标使用人群：**

- 部门业务人员与负责人（收入征收、资产、成本管理、合规、审计、财务报告、规划、预算执行、财务绩效分析等）；
- 财务与预算条线的管理者 / 决策层（查看绩效、偏差、加速计划等结论与建议）；
- 演示与售前场景中的客户方评审人员。

登录页提供按角色（分析师 / 经理等）进入的入口，不同角色进入的是同一套演示环境。

**整体功能简述：**

- **统一部门工作台**：覆盖 G-02～G-06 各业务总局下的部门，结构一致——职责概览 → 关键指标（KPI 轮播）→ 业务广场 → 智能助手（编排对话 + 多智能体流程入口）。
- **编排对话（Orchestrator Chat）**：输入业务请求，观看智能体时间线运行，最终进入人工审批（批准 / 退回 / 查看差异）。
- **智能查询（Smart Query）**：每个工作台右上角的浮动助手，在该部门作用域内即问即答。
- **业务广场（Business Plaza）**：以节点图呈现该业务分组的用例与跨部门数据流，区分「支持一键直达」与「仅上下文」两类用例。
- **多智能体流程页（Multi-Agent Flow）**：按总局绘制的数据源 → 用例链 → 人工关卡 → 可交付成果的完整编排图。
- **财务绩效应用（嵌入）**：集成的财务绩效分析 / 收入征收 / 预算执行视图，提供更深入的指标、地图与报表。
- **三语支持**：中 / 英 / 阿（阿拉伯语 RTL）全程切换。

## 二、输入输出规则说明

### 1. 用户侧可输入的内容类型、输入格式、输入限制

Demo 中用户的输入入口主要有三类：

**（1）编排对话输入框（Orchestrator Chat）**

- 输入类型：自然语言业务请求（如「解释为何执行率为 64%（目标 95%），识别滞后的阿玛纳，并起草供领导复核的加速说明」）。
- 输入格式：多行文本框（textarea），进入页面时预填该部门的默认请求，可自由编辑或清空重写。
- 提交方式：点击 **SEND** 按钮，或按 **Enter**（Shift+Enter 换行）。
- 输入限制：内容为空 / 全空白时 SEND 按钮禁用、不触发；运行（running）状态下输入框禁用、不接受新提交，需等本轮结束。
- 辅助输入：空状态下提供「建议请求」按钮，点击后将「标题 · 说明」填入输入框，用户可再编辑后发送。

**（2）智能查询输入框（Smart Query）**

- 输入类型：该部门作用域内的自然语言问题（如「为何执行率仅 64%」「哪些阿玛纳支出滞后」）。
- 输入格式：单行输入框；并提供「建议问题」列表可一键提问。
- 提交方式：点击 **发送**，或按 **Enter**。
- 输入限制：空 / 全空白不触发；正在思考（thinking）时不接受新提问；作用域标注为「只读」，仅做问答，不产生写操作。

**（3）页面控件类输入（非自由文本）**

- 登录页：角色下拉选择、验证码输入框（最多 6 位）、密码框（演示中为占位、只读）。
- 各页面：业务广场节点点击、KPI 轮播切换、语言切换、下一步动作按钮等（详见第三章）。

> 说明：Demo 不接受文件上传、不接受真实凭证；密码与验证码为演示占位，不做真实校验。

### 2. AI 端输出结果形式、输出内容结构、返回字段 / 信息维度

**（1）编排对话输出（核心）**

提交请求后，AI 端按「思考 → 时间线 → 人工审批」分阶段返回，结构如下：

- **思考阶段（Thinking）**：逐条点亮四步推理——解析意图 / 检查权限范围 / 选择智能体 / 规划子任务。
- **智能体时间线（Agent timeline）**：逐个执行的智能体步骤，每步包含字段：
  - 用例编号（如 UC-01 / UC-06 / UC-02，默认隐藏、可经 URL 开启显示）；
  - 步骤标题（如「拉取统一数据集」「计算执行与偏差」）；
  - 状态（排队 queued / 思考中 thinking / 完成 done）；
  - 结果摘要 / 度量（如「执行 64% · 较计划 −4.86B」「扫描 5 阿玛纳 · 2 个 <62%」）。
- **人工审批（Human-in-the-loop Review）**：包含审批说明正文、以及三个操作——批准 / 退回修正 / 查看差异；查看差异时展示新增（+）与移除（−）的逐行对比。
- **下一步动作（Next actions · owners）**：可执行动作清单，每项字段为：动作描述、负责人姓名、岗位角色、联系电话（可点击拨号）。
- **状态标识**：对话头部实时显示阶段标签（就绪 / 运行中 / 待审批 / 已完成 / 已退回）。

**（2）智能查询输出**

- 输出形式：对话气泡式问答；先显示「正在输入」动画（约 0.8 秒）后给出回答。
- 输出内容：针对该作用域的文本回答；同时在底部「智能体日志」记录一条查询。

**（3）KPI 与页面数据输出**

- 关键指标卡：字段为指标名、数值、口径说明（如「支出率 64.0% · 已花 11.12B ÷ 预算 17.37B」）；进入页面时数字有动态计数动画。
- 业务广场节点：字段为用例标题、参与智能体（概览显示「N agents」，展开显示具体智能体名）、跨部门数据流标签。
- 多智能体流程页：数据源 → 各用例（含智能体与输入/输出）→ 人工关卡 → 可交付成果（含下游消费方）。

### 3. 异常输入场景下的兜底返回规则

- **空 / 纯空白输入**：编排对话与智能查询均不触发，发送按钮禁用，页面无变化（不报错、不产生空响应）。
- **运行中重复提交**：运行 / 思考状态下忽略新的提交，避免并发；需等本轮结束或经审批进入下一状态。
- **审批退回（Return for fix）**：用户对 AI 草稿不满意时点击「退回修正」，AI 端返回「已退回修正」状态与说明，不发布结果，等待重新发起。
- **嵌入页加载失败**：嵌入的财务绩效应用若渲染异常，由错误边界兜底，显示「此分析视图渲染失败」的友好提示与错误详情，而非整页崩溃。
- **静态资源缺失**：登录页 Logo / 配图等按多路径回退，最终缺失时隐藏，不阻塞页面。
- **登录校验**：演示环境不校验密码 / 验证码，点击登录即以所选角色进入（合成数据，无真实鉴权）。

## 三、页面布局与界面呈现说明

### 1. 整体页面结构、分区布局

登录后进入主框架，自上而下 / 自左而右分为：

- **顶部栏（TopBar）**：品牌 Logo 与名称、语言切换、通知铃铛、用户菜单 / 退出。
- **左侧导航（Sidebar）**：按部门 / 总局组织的手风琴式菜单，用于在各工作台与功能页之间切换。
- **主内容区（Content）**：渲染当前路由对应的页面（工作台 / 流程页 / 报告中心 / 嵌入应用等）。
- **底部智能体日志（Agent Log）**：滚动显示系统与智能体活动；右下角为构建版本号入口，点击打开「更新日志」。

**部门工作台（DeptWorkspace）内部分区**（全部门统一）：

- 顶部头区：右上浮动「智能查询」按钮、部门标题与职责说明、右侧「G-06 故事线 · 下游演进」链路条；
- 关键指标区：KPI 轮播（多屏指标）；
- 下方两栏：左栏为「业务广场」，右栏为「编排对话」+「多智能体流程入口卡」。

### 2. 各功能按钮、控件名称、位置及展示样式

- **登录页**：角色下拉、密码框（含显示/隐藏「眼睛」图标）、验证码（6 位 + 刷新按钮）、「登录」主按钮、Nafath（国家统一身份）登录按钮、底部语言切换。
- **顶部栏**：语言切换按钮（中/英/阿循环）、通知铃铛（带未读数）、用户头像菜单。
- **编排对话区**：头部阶段标签；建议请求按钮（空状态）；底部输入框 + **SEND** 按钮；审批区的 **✓ 批准** / **↺ 退回修正** / **⌥ 查看差异** 三个按钮；下一步动作每项右侧的 **📞 电话** 链接。
- **智能查询面板**：浮动 **🤖** 触发按钮（工作台右上）；面板含作用域标注、问答区、「建议问题」可展开列表、输入框 + **发送** 按钮、关闭 **✕**。
- **业务广场**：节点卡（显示用例标题 + 智能体数）；「支持一键直达」的节点右下角有 **↗** 一键打开按钮，「仅上下文」节点显示「context」标签且不可直达；点击节点可选中并显示跨部门连线；可展开为全屏大图。
- **多智能体流程页**：返回按钮 **‹**；流程画布（数据源框 → 用例节点含智能体标签 → 人工关卡 → 可交付成果列）。
- **KPI 轮播**：多屏指标卡，支持切换；数值带计数动画。

### 3. 不同状态下页面展示差异（加载中、等待响应、返回结果、报错状态）

- **加载中**：嵌入的财务绩效应用懒加载时显示「Loading…」占位；KPI 数字以计数动画呈现。
- **等待响应（运行中）**：编排对话头部显示「运行中」并带脉冲点；输入框与 SEND 禁用；先显示思考四步逐条点亮，再显示智能体时间线逐步从「排队 → 思考中 → 完成」。智能查询显示「正在输入」三点动画。
- **返回结果**：编排对话进入「待审批」，展示审批说明与三个操作按钮；批准后变「已完成」并显示「下一步动作」；智能查询追加回答气泡。
- **报错 / 兜底状态**：嵌入页渲染失败显示错误边界提示；空输入时按钮禁用无响应；退回修正显示「已退回修正」红色状态块。

## 四、业务操作流程

### 1. 用户完整操作步骤时序流程

以「财务绩效分析部」为典型场景：

1. 在登录页选择角色 → 点击登录，进入主框架（默认落地页 / Hub）。
2. 通过左侧导航进入目标部门工作台。
3. 浏览顶部职责说明、KPI 指标与「G-06 故事线」链路，建立全局认知。
4. 在「业务广场」查看本组用例与跨部门数据流；对「支持一键直达」的用例点击 **↗** 跳转到对应页面（如绩效应用、报告中心、告警等）。
5. 在右栏「编排对话」中编辑或直接使用预填请求 → 点击 **SEND**。
6. 观看 AI 思考四步 → 智能体时间线逐步完成。
7. 在「人工审批」环节选择：**批准**（生成结论与下一步动作）/ **退回修正** / **查看差异**。
8. 批准后查看「下一步动作 · 负责人」，必要时点击电话联系负责人。
9. 如需即问即答，点击右上 **🤖** 打开「智能查询」，在作用域内提问或选建议问题。
10. 可随时切换语言、查看通知与告警、打开「更新日志」。

### 2. AI 内部执行逻辑流程（从接收输入到生成结果的完整链路）

> 注：当前 Demo 的「AI 执行」为按固定时序编排的脚本化演示，用于真实还原编排过程的形态。

1. **接收输入**：捕获用户请求文本，记录到智能体日志，置为「运行中」。
2. **意图与权限**：依次完成解析意图 → 检查权限范围 → 选择智能体 → 规划子任务（思考四步）。
3. **多智能体执行**：按编排时间线依次调度各智能体（如统一数据 → 计算执行与偏差 → 检测滞后 → 起草说明），每步产出度量摘要并更新状态。
4. **生成草稿**：汇总为面向决策的结论与草稿（如加速说明），进入「待审批」。
5. **人工关卡**：等待用户批准 / 退回；批准则定稿并生成「下一步动作 + 负责人」；退回则置为「已退回」、不发布。
6. **跨用例衔接**：结果按业务故事线流向下游用例（绩效与告警 → 资产 → 成本与合规 → 报告与智能查询）。

## 五、页面跳转与交互关联关系

### 1. 各页面 / 弹窗之间的跳转触发条件、跳转先后顺序

- **登录 → 主框架**：登录页点击「登录」或「Nafath」后以所选角色进入，渲染主框架与默认页。
- **导航跳转**：左侧 Sidebar 选择部门 / 功能，主内容区切换到对应路由（各部门工作台、多智能体流程页、报告中心、嵌入应用等）。
- **业务广场一键直达**：点击节点 **↗** 按当前用例跳转——告警抽屉、绩效应用、报告中心、成本与资金、合规裁定、收入 / 资产工作台等；跳转前记录来源页用于返回。
- **流程入口卡**：工作台右栏「多智能体流程入口卡」跳转到对应总局的流程页。

### 2. 按钮点击、指令发送后对应的页面变化、弹窗唤起逻辑

- **SEND / Enter**：触发编排运行，对话区依次渲染思考、时间线、审批；不跳转页面，原地演进。
- **批准 / 退回 / 查看差异**：原地切换审批结果区；查看差异为就地展开 / 收起，不跳转。
- **🤖 智能查询**：唤起 / 关闭浮动问答面板（覆盖于当前页之上），不离开当前工作台。
- **通知铃铛 / 告警**：唤起通知列表或告警抽屉（覆盖层），点击空白处关闭。
- **更新日志**：点击底部版本号唤起「Release Notes」弹窗，点击遮罩或 ✕ 关闭。
- **下一步动作电话**：点击 📞 触发拨号链接，不影响页面状态。

### 3. 页面返回、刷新、重新发起请求后的状态变化规则

- **返回**：流程页 / 嵌入页的「返回」按钮回到记录的来源页（若有），否则回到该部门默认页；一键直达进入的嵌入页返回时恢复宿主路由。
- **刷新**：Demo 状态存于内存，刷新后回到初始（登录 / 默认页），对话、问答、审批等会话内状态会重置。
- **重新发起请求**：在「已完成 / 已退回」后再次编辑并 SEND，将清空上一轮时间线与审批，重新从思考阶段开始。
- **语言切换**：即时切换中 / 英 / 阿并应用 RTL 方向，不丢失当前所在页面。

## 六、补充说明

### 1. 当前 Demo 已实现能力范围

- 覆盖 G-02～G-06 各业务总局的 **统一部门工作台**（财务绩效分析、规划、预算执行、财务权益、审计、会计、合规、成本管理、财务报告，以及收入征收、资产）。
- **编排对话**全流程：自然语言请求 → 思考 → 智能体时间线 → 人工审批（批准 / 退回 / 查看差异）→ 下一步动作（负责人 + 电话）。
- **智能查询**浮动助手：作用域内即问即答与建议问题。
- **业务广场**：分组用例节点图、跨部门数据流连线、「一键直达 / 仅上下文」区分、全屏展开。

### 2. 暂未实现 / 当前为演示占位的功能

- **真实 AI 推理**：编排时间线与智能查询回答为脚本化 / 预置内容，未接入真实大模型与实时计算。
- **真实数据接入**：全部为合成数据，未连接真实源系统（SAP、Esnad 等仅为展示标签）。
- **真实鉴权与安全**：登录、密码、验证码、Nafath 均为演示占位，不做真实校验，无角色级权限控制。
- **写操作与落账**：审批后的「批准 / 下一步动作」不会真正发文、落账或下发到外部系统。
- **持久化**：无后端存储，会话内状态在刷新后重置，不保存历史记录。
- **报表 / 文件导出**：未提供真实导出、打印或附件上传能力。
- **通知拨号等外部联动**：电话为 tel 链接示意，通知 / 告警为静态演示数据。

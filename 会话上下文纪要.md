## 三、Workspace 设计理念

所有部门共用一套 `DeptWorkspace` 模板，由 cfg 配置对象驱动——**结构统一、内容按部门替换**。每个工作台是同一套外壳与交互：

```
顶部（职责标题 + 浮动智能查询 + G-06 故事线条）
  → KPI 轮播
  → 两栏：左 业务广场 | 右 编排对话 + 多智能体流程入口卡
```

编排对话走统一状态机：运行 →（思考四步 → 智能体时间线）→ 人工审批（批准 / 退回 / 查看差异）→ 下一步动作（含负责人 + 电话）。同一总局下的子部门共享该总局的一套多智能体流程。一句话：**一个模板，各部门只换 KPI / 广场 / 流程 / 剧本。**

## 四、关键决策与原则（重要）

- **「分析后套进模板」**（多次强调）：拿到设计图是**分析其内容、装入既有模板**（DirectorateFlow 节点链 / 两栏业务广场），**不是照图重做新布局**。期间几次误做成新组件（OrchestrationPlaza、LayeredFlow）后均已回退。
- **数据口径以 BRD 为准**；与 UC-06 重叠处以 **UC-06 为唯一真值**（如：修订预算 17.37B、已花 11.12B、执行率 64.0%、剩余 6.25B、计划至今 15.98B、偏差 −4.86B）。
- **默认隐藏 UC-XX / G- 代号**，对外展示 BRD 名称；`?uc=true` 可显示代号（`SHOW_UC` 控制）。
- **图片 vs BRD 冲突**时：图片驱动的流程以图片为准，并标注与 BRD 的偏差（如个别图含 Hyperion/MTFP 而 BRD 已移除）。
- **G-06 故事线**：UC-01 统一数据 → UC-13 收入/征收/排除项（核心）→ UC-06 / UC-02 绩效与告警 → UC-14 资产与资本化 → UC-12 / UC-11 成本与合规 → UC-10 / UC-03 报告与智能查询；Assets 工作台复用同链、焦点在 UC-14。
